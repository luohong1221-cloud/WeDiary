1) 产品定位与PRD一键生成 Prompt（产品经理视角）

把下面整段丢给 AI（ChatGPT / Claude 都行）：

Prompt：

你是资深移动端产品经理，请设计一款 Android 日记 App（对标 Journey / Day One / Diaro 等主流日记应用形态），要求：

支持不限量创建日记；

每篇日记可上传多张图片（支持相册选择/拍照/图片预览/删除/排序）；

强调“快速记录 + 富媒体 + 隐私安全 + 强搜索”。
请输出：

产品一句话定位、目标用户、核心场景；

关键功能清单（Must/Should/Could）；

10 条用户故事（含验收标准）；

信息架构 IA（模块与层级）；

版本规划（MVP -> V1.0 -> V1.5），并说明每期取舍逻辑；

关键指标（激活、留存、记录频次、搜索使用率、图片使用率、备份开通率等）与埋点建议。
输出要具体可落地，适合直接给研发与UI执行。

2) 核心页面与交互稿 Prompt（UX / 交互设计）

Prompt：

你是资深 Android UX 交互设计师，请为一款“富媒体日记App”设计关键页面与交互细节（对标 Journey / Day One 的清爽记录体验与富媒体编辑能力）。必须包含：

首页（时间线/卡片流）

日历视图（按天查看、当天多条）

新建/编辑日记（富文本输入 + 图片网格 + 大图预览）

日记详情页（沉浸阅读）

搜索页（全文检索、按标签/日期/心情筛选）

标签管理页

设置与隐私（PIN/指纹锁、导出/备份）
请输出：

每个页面的布局结构（顶部栏、主内容、底部操作）

关键交互（新增图片、图片排序、删除确认、保存草稿、退出防丢、空状态）

组件清单（RecyclerView 卡片、BottomSheet、Chip、DatePicker、ImageViewer 等）

错误态/加载态/权限弹窗文案建议（相册权限、相机权限、存储空间不足等）

新手引导方案（3步以内）
用 Android 设计习惯与 Material You 风格描述，尽量具体。

3) UI 视觉风格 Prompt（给“生成式UI/风格板”）

如果你用的是 Figma AI、某些 UI 生成器，建议用这种“风格+组件库+状态”描述：

Prompt：

生成一套 Android 日记App UI 视觉稿（Material You 风格，温暖克制、留白充足、适合长文阅读）。
设计包含：

亮色/暗色模式

首页卡片流（每条：日期、标题、摘要、图片缩略图、标签chips）

编辑页（顶部保存、正文输入、图片网格、添加图片按钮、底部工具条）

日历页（顶部月份切换、日期点标记有内容的天）

搜索筛选（搜索框、筛选chips、结果高亮）
细节要求：

文字排版适合长阅读（标题/正文/元信息层级明显）

图片缩略图圆角、可见阴影层次

空状态插画风简洁

组件状态齐全：默认/按下/禁用/加载骨架屏
输出：色板、字体层级、间距体系、关键组件样式与 3 张核心页面视觉稿。

4) “不限量 + 图片”数据与本地存储设计 Prompt（Android 架构）

你要做“不限量”，本地优先（Offline-first）是主流路线；图片通常存本地文件/私有目录，数据库只存 URI/路径与元信息。

Prompt：

你是 Android 架构师，请为“离线优先”的日记App设计数据模型与存储方案，核心要求：不限量日记、每篇多图、可全文检索、可按日期/标签筛选。
技术栈：Kotlin + Room + MVVM（或 MVI）+ Jetpack Compose（或 XML 也可）。
请输出：

Room 表结构（DiaryEntry、DiaryImage、Tag、EntryTagCrossRef 等）字段与索引设计（重点：按日期排序、全文检索FTS、按标签查询）

图片存储策略（私有目录、压缩与多尺寸缓存、删除一致性、备份考虑）

Repository 层接口（新增日记、添加图片、图片排序、删除、搜索）

性能与容量策略（分页、缩略图、图片懒加载、数据库增长）

数据安全（加密方案可选：数据库加密/图片加密/应用锁）
用清晰的结构化说明，不要泛泛而谈。

5) 图片上传与编辑体验 Prompt（把交互做“像主流App”）

Prompt：

你是移动端体验设计专家，请设计“日记内多图管理”的最佳实践交互（对标主流日记App的流畅体验）。场景：用户写一篇日记，插入 1~30 张图片。
请给出：

添加入口（相册/拍照/最近使用）与推荐默认行为

图片展示（网格、瀑布流、横向胶片条三种方案对比，推荐一种）

图片预览（全屏滑动、缩放、下载/分享是否需要）

排序（长按拖拽）与删除（撤销toast）

上传/处理（压缩、生成缩略图、后台处理）时的进度与失败重试

细节：写作不中断（键盘不乱跳）、保存草稿、退出防丢
输出可直接交付开发的交互规格点与状态机。

6) 文案与引导 Prompt（让产品更像“成熟商业App”）

Prompt：

你是资深产品文案，请为 Android 日记App 输出一套中文文案：

首次启动 3 屏引导（每屏标题+一句话说明）

空状态文案：首页空、日历空、搜索无结果、标签空

权限弹窗前置说明：相册权限、相机权限（解释用途、用户收益）

删除/清空/退出未保存的确认弹窗文案（含危险操作提示）

隐私与安全卖点文案（应用锁、备份、导出）
风格：温柔、克制、有安全感，不营销夸张。

7) 你可以直接采用的“功能对标清单”（做原型时照抄）

主流日记App普遍会有这些模块（能显著提升“像成熟产品”的感觉）：

时间线（按天/周/月） + 日历入口

标签 / 搜索（全文）

图片、地点、天气/心情（可选）

应用锁（PIN/指纹）

导出（PDF/Markdown/纯文本）与备份（本地/云端可选）
这些方向也符合媒体丰富与“记录习惯”产品路线。
